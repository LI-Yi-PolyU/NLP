# StoryWeaver 游戏剧本（可演示版）

## 1. 核心设定
- 背景：中古奇幻世界，青石村被未知阴影笼罩。
- 主角：玩家（外来者）。
- 关键NPC：老村长（elder）、猎人阿洛（hunter）。
- 关键物品：火把（torch）、锈剑（sword）。

## 2. 主线目标
玩家需要在有限回合内完成“调查 -> 获取关键物品 -> 返回村庄”的闭环，稳定村庄局势。

主线成功判定（当前实现）：
- 玩家持有 sword；
- 玩家回到 village。

## 3. 推荐演示流程（8-12回合）
1. 开场（village）
- 玩家观察村庄，和老村长对话，得知森林与洞穴的风险。

2. 获取照明（village）
- 玩家执行“拿起火把”，建立基础探索能力。

3. 前往森林（forest）
- 玩家与猎人阿洛交谈，获知洞穴危险和旧传闻。

4. 深入洞穴（cave）
- 玩家使用火把探索洞穴环境，找到锈剑线索。

5. 拿到锈剑（cave）
- 玩家执行“拿起锈剑”，完成关键物品收集。

6. 返回村庄（village）
- 玩家回到村庄触发主线完成结局。

## 4. 分支设计
### A. 稳健线（推荐课堂演示）
- 行为特点：多问讯息、少冒险冲突。
- 优点：一致性冲突少、叙事稳定。

### B. 风险线
- 行为特点：快速探索 + 频繁使用物品/攻击。
- 优点：戏剧冲突强。
- 风险：更容易触发规则冲突与负面结局。

### C. 交易线
- 行为特点：偏重与NPC协商，换取信息优势。
- 优点：更强调“叙事推理”和关系演化。

## 5. 终局与文本风格
当前系统支持三类终局：
1. 胜利结局（victory）
- 文案名：守望者归来
- 触发：持有 sword 且位于 village

2. 超时结局（timeout）
- 文案名：未竟之夜
- 触发：达到最大回合上限（默认20）仍未完成主线

3. 失败结局（bad_forbidden_attack）
- 文案名：秩序崩坏
- 触发：玩家攻击关键角色 elder

## 6. 演讲用“30秒摘要”
在 StoryWeaver 中，玩家从青石村出发，通过对话与探索获取火把和锈剑，再返回村庄完成主线。系统用本地NLU识别意图、状态数据库维护世界事实、一致性规则阻止非法行动，并由生成模型实时输出剧情。最终会根据玩家行为落入胜利、超时或失败三类结局，实现可解释、可终止、可演示的文本冒险闭环。

## 7. 同义词表配置（可扩展性亮点）
系统支持通过配置文件扩展口语表达，不需要改代码。

配置文件：
- config/entity_aliases.yaml

当前支持三类实体：
- location：地点口语词（例如 洞穴/森林/村庄）
- item：物品口语词（例如 锈剑/火把）
- character：角色口语词（例如 老人/猎人）

使用方式：
1. 在对应 canonical id 下添加同义词。
2. 保存后重启应用。
3. 新增口语输入将自动映射到标准实体ID。

示例：
```yaml
location:
	cave: ["洞穴", "山洞", "岩洞"]

item:
	sword: ["锈剑", "古剑", "短剑"]

character:
	elder: ["老村长", "长者", "老人"]
```

演讲要点：
- 语言鲁棒性：支持自然口语输入，不要求用户记忆系统ID。
- 可维护性：策划可直接改YAML扩词，无需工程改动。
- 可迁移性：换世界观时只需重配实体词表即可复用引擎。
