# StoryWeaver 项目技术报告

## 1. 项目概述

StoryWeaver 是一个面向课程场景的 AI 文本冒险系统，目标是把“自然语言理解 -> 世界状态更新 -> 一致性约束 -> 剧情生成 -> 自动评测/证据导出”整合为可运行、可演示、可复现的一体化链路。

系统支持两种运行模式：
- LIVE：真实引擎链路，强调真实效果与可解释性。
- SCRIPTED：关键步骤优先走缓存，强调演示稳定性与低风险。

项目入口与核心脚本：
- app.py：Gradio 前端入口。
- run_data_pipeline.py：离线数据构建与语料构建。
- run_evaluation.py：自动评测入口。
- run_day3_evidence.py：答辩证据导出入口。

## 3. 模型与算法设计

### 3.1 意图识别模型（NLU）
实现位置：src/nlu/local_bert.py

采用“多级回退”策略，保证可用性与稳定性：
1) 线性模型优先：
- 使用 data/benchmarks/intent_train.jsonl 训练字符级 TF-IDF + 逻辑回归。
- 若置信度 >= 0.55，直接输出。

2) 本地 Transformer 分支（可用时）：
- 配置项 local.bert_model，默认 bert-base-chinese。
- 使用 AutoTokenizer + AutoModelForSequenceClassification（5 类意图）。
- 若分类头置信度低，会触发关键词纠偏。

3) 关键词规则回退：
- 基于中英关键词词表进行意图判定。
- 未命中时输出 UNKNOWN。

4) API 回退（低置信触发）：
- 当本地意图置信度低于阈值（GameEngine 中默认 0.7）且配置了 DEEP_SEEK_API_KEY，调用 src/nlu/api_fallback.py。
- API 回退模型默认使用 deepseek-chat（model_nlu_fallback）。

### 3.2 实体抽取
实现位置：src/nlu/entity_extractor.py

- 基于 world_setting.yaml 的地点/角色/物品 ID 与中文名做规则匹配。
- 采用长词优先与跨度去重，降低重叠实体误判。
- 在 GameEngine 中配合 entity_aliases.yaml 做别名归一。

### 3.3 一致性校验模型与规则
实现位置：src/core/consistency_checker.py

一致性采用三层策略：
1) 硬规则层（hard constraints）：
- dead_character_cannot_speak
- item_not_held_cannot_be_used
- location_not_connected_cannot_travel

2) 数据库状态冲突层：
- 基于事实库历史状态，阻止非法状态回退等冲突。

3) 语义层（可选 NLI）：
- 若本地 NLI 模型可用（默认 uer/roberta-base-chinese_nli），对同主体状态做语义冲突补充验证。

自动修复策略：
- 对“未持有物品却使用”可自动修复为安全动作（fumble_no_item），避免流程中断。

### 3.4 剧情生成模型（NLG）
实现位置：src/generation/story_gen.py

- 生成模型：deepseek-chat（model_generation）。
- 接口协议：OpenAI 兼容 chat.completions。
- 输出约束：优先使用 JSON Schema（config/prompts/story_generation.json），不支持时退回 json_object。

生成后处理：
- 输出结构标准化（narration/dialogue/state_changes/next_options/consistency_notes/rag_sources）。
- 中文强制修正（防止英文输出）。
- 选项去重与补全。
- 二次一致性验证并附修正提示。

### 3.5 检索增强（RAG）
实现位置：src/generation/retriever.py

- 语料来源：data/raw_corpus/*.jsonl（由 run_data_pipeline.py 构建）。
- 向量模型：config 中 local.embedding_model，默认 shibing624/text2vec-base-chinese。
- 检索引擎：
  - 优先 FAISS 内积索引。
  - 无 FAISS 时回退 numpy 相似度。
  - 无 sentence-transformers 时回退哈希向量嵌入。
- 检索结果含 location boost，提升同地点剧情相关性。

## 4. 系统架构与执行流程

### 4.1 离线流程（数据与索引准备）
入口：run_data_pipeline.py

1) 读取 world_setting.yaml 与 model_config.yaml。
2) 生成意图数据集：intent_train.jsonl / intent_test.jsonl。
3) 生成一致性测试集：consistency_test.jsonl。
4) 构建叙事语料：synthetic_scenarios.jsonl（模板或 API 生成）。
5) 由检索模块构建向量索引：narrative.index / narrative_vectors.npy / narrative_meta.json。

### 4.2 在线回合流程（核心链路）
核心调度：src/core/game_engine.py

单回合处理步骤：
1) 接收玩家输入。
2) NLU 输出意图与实体（必要时 API 回退）。
3) 结合规则护栏进行意图纠偏（如拾取类动作）。
4) 将意图映射为事实提议（Fact 列表）。
5) 一致性校验：通过/自动修复/拒绝并备注。
6) 状态更新到 StateManager（SQLite 事实库）。
7) 调用 StoryGenerator：
- RAG 检索相关场景。
- 生成结构化剧情与选项。
- 二次一致性与中文化后处理。
8) 返回结构化输出给前端。
9) 记录调试信息与轨迹（DemoController）。

### 4.3 演示控制与可视化
- DemoController 提供 LIVE 与 SCRIPTED 双模式。
- SCRIPTED 模式可对关键步骤使用缓存响应，降低现场不可控风险。
- app.py 使用 Gradio 进行交互式展示，包含剧情区、状态区、调试视图。

## 5. 状态管理与数据资产

### 5.1 状态管理
实现位置：src/core/state_manager.py

- 使用 SQLite facts 表持久化事实。
- 关键字段：subject、predicate、object、turn、valid。
- 支持世界初始化、状态快照、按事实增量更新、世界重置。

### 5.2 关键数据目录
- data/benchmarks：评测集与轨迹。
- data/raw_corpus：RAG 语料。
- data/vector_index：向量索引产物。

## 6. 评测体系与当前结果

### 6.1 自动评测指标
实现位置：src/evaluation/auto_eval.py

覆盖四项：
1) intent_recognition：意图识别准确率/F1。
2) consistency_detection：一致性检测准确率/F1。
3) generation_latency：端到端回合平均延迟（GameEngine 口径）。
4) branch_diversity：叙事分支多样性（基于文本相似度）。

### 6.2 当前评测快照（evaluation_report.json）
- 意图识别 accuracy = 1.0，f1_macro = 1.0
- 一致性检测 accuracy = 1.0，f1_macro = 1.0
- 端到端平均延迟 avg_ms = 8.68（12 轮）
- 分支多样性 diversity_score = 1.0

说明：以上分数来自当前数据分布与评测集，代表“项目内口径表现”。若用于更强泛化对比，建议补充跨域样本与人工评测协议。

## 7. 容错设计与工程鲁棒性

1) 多级回退：
- 本地线性模型 -> 本地 Transformer/关键词 -> API fallback。

2) 生成容错：
- JSON Schema 不可用时自动退回 json_object。
- API 不可用时可走模板/兜底叙事路径。

3) 一致性保护：
- 冲突动作可拒绝或自动修复，降低状态污染风险。

4) 演示稳定性：
- SCRIPTED 缓存与 FALLBACK 机制保障答辩可控。

## 8. 项目亮点

1) 端到端链路完整：从输入到评测到证据导出形成闭环。
2) 结构化输出设计：叙事、状态变更、选项、备注可直接驱动前端与分析。
3) 多重稳态策略：本地优先、API 增强、规则兜底的混合架构。
4) 可解释性较好：每轮可查看意图来源、规则结果、检索来源。

## 9. 局限与优化建议

1) 数据分布风险：
- 当前意图与一致性评测以合成/规则数据为主，存在同分布偏高可能。

2) 模型能力边界：
- 本地 Transformer 分类头未显式微调流程，实际效果较依赖线性模型与规则回退。

3) 长程叙事记忆：
- 回合变长后，状态压缩与检索上下文组织仍有优化空间。

4) 评测口径扩展：
- 建议补充人工评测维度：可玩性、叙事连贯性、分支区分度、角色一致性。

5) 工程化增强：
- 增加单元测试/回归测试、结构化日志、错误分级与可观测性仪表。

## 10. 结论

StoryWeaver 已实现课程要求下的完整技术闭环：
- NLU：可解释、可回退的意图理解与实体抽取；
- Consistency：规则+语义的多层状态一致性防线；
- NLG：RAG 增强的结构化剧情生成；
- Evaluation：自动评测与答辩证据一键导出。

作为课程项目，该系统已具备较高完成度与演示可用性。若继续演进到更高质量版本，重点应放在：跨域泛化评测、长程叙事稳定性和工程可观测性。
